* SQL이란 무엇인가요?
  
  SQL은 Structured Query Language의 약어로, 데이터베이스를 관리하기 위해 사용되는 프로그래밍 언어입니다. SQL은 데이터를 검색, 삽입, 업데이트 및 삭제하는 데 사용되며, 대부분의 관계형 데이터베이스 시스템에서 지원됩니다.
* 관계형 데이터베이스란 무엇인가요?
  
  관계형 데이터베이스는 데이터를 테이블 형태로 저장하고 관리하는 데이터베이스입니다. 이러한 데이터베이스에서는 데이터를 테이블로 분리하고, 테이블 간의 관계를 정의하여 데이터를 구성합니다.
* SQL에서 Join의 종류는 어떤 것이 있나요? 각각의 차이점은 무엇인가요?

  SQL에서 Join은 두 개 이상의 테이블에서 데이터를 가져오는 방법입니다. 

  1. INNER JOIN

      INNER JOIN은 두 개 이상의 테이블에서 공통된 값을 가진 레코드를 선택하는 Join입니다. INNER JOIN을 사용하면 조인 대상 테이블에서 모두 일치하는 레코드만을 가져옵니다.

  2. LEFT JOIN

    LEFT JOIN은 왼쪽 테이블의 모든 레코드를 선택하고, 오른쪽 테이블에서 왼쪽 테이블과 공통된 값을 가진 레코드를 선택하는 Join입니다. 오른쪽 테이블에 일치하는 레코드가 없는 경우 NULL 값을 반환합니다.

  3. RIGHT JOIN

    RIGHT JOIN은 LEFT JOIN과 반대로 오른쪽 테이블의 모든 레코드를 선택하고, 왼쪽 테이블에서 오른쪽 테이블과 공통된 값을 가진 레코드를 선택하는 Join입니다. 왼쪽 테이블에 일치하는 레코드가 없는 경우 NULL 값을 반환합니다.

  4. FULL OUTER JOIN

    FULL OUTER JOIN은 왼쪽 테이블과 오른쪽 테이블 모두에서 공통된 값을 가진 레코드를 선택하는 Join입니다. 이 Join은 왼쪽 테이블과 오른쪽 테이블에서 일치하지 않는 레코드도 모두 선택합니다. 따라서 FULL OUTER JOIN을 사용하면 레코드가 일치하지 않는 경우에도 결과를 반환할 수 있습니다.

  5. CROSS JOIN
    CROSS JOIN은 두 개 이상의 테이블에서 모든 가능한 조합을 선택하는 Join입니다. 이 Join은 조인 대상 테이블의 모든 레코드를 모두 조합하여 결과를 반환합니다.

  
* SQL에서 Group By와 Having의 차이점은 무엇인가요?

  SQL에서 GROUP BY와 HAVING은 데이터를 그룹화하고 조건에 따라 필터링하는데 사용되는 구문입니다. 그러나 두 구문의 차이점은 GROUP BY는 특정 열을 기준으로 데이터를 그룹화하는 데 사용됩니다. GROUP BY는 그룹 함수(SUM, COUNT, AVG 등)와 함께 사용됩니다. HAVING은 그룹화된 데이터에서 조건에 따라 필터링하는 데 사용됩니다. HAVING은 WHERE 구문과 비슷하지만, WHERE 구문은 단일 행에서 조건을 검사하는 반면, HAVING 구문은 그룹화된 결과에서 조건을 검사합니다.

* SQL에서 Index란 무엇인가요? 어떤 경우에 사용하나요?

  SQL에서 Index는 데이터베이스 테이블의 특정 열(또는 열의 조합)에 대한 검색 속도를 높이기 위해 사용되는 데이터 구조입니다. Index는 특정 값을 가진 레코드를 빠르게 찾기 위해 사용됩니다.

  1. 검색 속도 향상
   
    Index를 사용하면 특정 열의 값을 가진 레코드를 빠르게 찾을 수 있습니다. Index를 사용하지 않으면 데이터베이스가 전체 테이블을 검색해야 하므로 검색 속도가 느려집니다.

  2. 정렬 속도 향상
   
    Index를 사용하면 데이터를 정렬할 때 빠른 속도를 보장할 수 있습니다. Index를 사용하지 않으면 정렬 작업에 많은 시간이 걸립니다.

  3. 중복 제거
   
    Index를 사용하면 중복된 레코드를 방지할 수 있습니다. Index를 사용하여 유일한 값을 가진 레코드를 유지할 수 있습니다.


* SQL에서 View란 무엇인가요? 어떤 경우에 사용하나요?

    SQL에서 View는 하나 이상의 테이블을 기반으로 생성된 가상 테이블입니다. View는 실제 데이터를 저장하지 않고, 저장된 테이블의 쿼리 결과를 참조하여 생성됩니다. View는 다른 테이블에서 데이터를 가져와 복잡한 쿼리를 단순화하고, 사용자가 데이터를 읽고 쓰는 데 쉽게 접근할 수 있도록 도와줍니다.
    
* SQL에서 Transaction이란 무엇인가요? 어떤 경우에 사용하나요?

  SQL에서 Transaction은 데이터베이스에서 수행하는 하나 이상의 작업 단위입니다. 하나의 Transaction은 일련의 쿼리 또는 명령문을 포함할 수 있으며, 이를 모두 실행하거나 모두 실행하지 않습니다. Transaction은 모든 작업이 완료되거나 실패하는 경우에만 완료됩니다.
  이를 통해 데이터의 일관성, 복구 기능, 동시성 제어등의 역할을 수행합니다.


* SQL에서 Subquery란 무엇인가요? 어떤 경우에 사용하나요?
  
    SQL에서 Subquery는 하나의 쿼리에서 또 다른 쿼리를 포함하는 쿼리입니다. Subquery는 외부 쿼리의 결과에 따라 내부 쿼리가 실행되며, 결과를 반환합니다.

* SQL에서 정규화란 무엇인가요? 어떤 경우에 사용하나요?
  
  정규화는 데이터베이스 설계의 기본 개념 중 하나로, 중복을 최소화하고 데이터의 일관성과 무결성을 유지하기 위해 데이터를 구조화하는 프로세스입니다. 

  - 1NF (제1정규화) : 각 컬럼이 원자 값(Atomic Value)을 갖도록 테이블을 구성합니다.
  - 2NF (제2정규화) : 테이블 내의 모든 컬럼이 완전 함수적 종속 관계에 있을 때 1NF를 만족하는 테이블을 더 이상 분리하지 않습니다.
  - 3NF (제3정규화) : 2NF를 만족하는 테이블에서 이행적 종속 관계를 제거하여 테이블을 더 이상 분리하지 않습니다.

* SQL에서 Triggers란 무엇인가요? 어떤 경우에 사용하나요?

  트리거(Trigger)는 데이터베이스에서 특정 이벤트가 발생할 때 자동으로 실행되는 작업입니다. 이벤트는 특정 테이블에서 데이터가 삽입, 수정 또는 삭제될 때 발생할 수 있습니다. 트리거는 일반적으로 데이터를 자동으로 갱신하거나 검사하기 위해 사용됩니다.

* SQL에서 LIKE와 IN의 차이점은 무엇인가요?
 
    LIKE 연산자는 문자열을 비교할 때 사용됩니다. 
    IN 연산자는 목록 안에 있는 값과 일치하는 모든 값을 반환합니다. 

* SQL에서 외래키(Foreign Key)란 무엇인가요? 어떤 경우에 사용하나요?

  외래키(Foreign Key)는 관계형 데이터베이스에서 두 개 이상의 테이블 간의 관계를 맺기 위한 기능입니다. 외래키는 하나의 테이블의 컬럼이 다른 테이블의 기본키(primary key)나 후보키(candidate key)에 대한 참조를 포함하는 제약 조건(constraint)입니다.
  이를 통해 데이터의 일관성, 데이터 무결성 보장, 데이터 연결 등의 사용됩니다.

* SQL에서 NULL값에 대한 이해가 있나요? NULL 값이 무엇인가요?

    SQL에서 NULL 값에 대한 이해가 있습니다.
    NULL 값은 데이터베이스에서 누락된 값, 알 수 없는 값, 혹은 존재하지 않는 값의 표현입니다. 즉, NULL은 아직 값이 할당되지 않았거나, 값이 없는 것을 나타냅니다. 

* SQL에서 인덱스를 생성할 때 주의해야 할 점은 무엇인가요?

    인덱스를 지나치게 많이 생성하지 않는 것이 중요합니다. 인덱스는 데이터베이스 쿼리의 성능을 향상시키지만, 인덱스가 많아질수록 데이터베이스의 수정 및 삽입 작업이 느려질 수 있습니다. 따라서, 인덱스를 생성할 때는 필요한 경우에만 생성해야 합니다.
    인덱스를 생성할 때는 데이터베이스의 특성에 맞게 인덱스를 생성해야 합니다. 예를 들어, 데이터베이스의 크기와 같은 요소를 고려하여 인덱스의 크기와 범위를 결정해야 합니다.

* SQL에서 DDL, DML, DCL이란 무엇인가요? 각각의 역할은 무엇인가요?

  - DDL(Data Definition Language): 데이터 정의 언어
  DDL은 데이터베이스의 스키마를 정의하고 변경하기 위해 사용됩니다.
  - DML(Data Manipulation Language): 데이터 조작 언어
DML은 데이터를 쿼리하거나 데이터베이스에서 데이터를 조작하는 데 사용됩니다. 
  - DCL(Data Control Language): 데이터 제어 언어
DCL은 데이터베이스 사용자의 권한을 제어하는 데 사용됩니다. 

* SQL에서 ACID 모델이란 무엇인가요? 각각의 개념에 대해 설명해주세요.

    ACID 모델은 데이터베이스 트랜잭션의 안전성과 일관성을 보장하기 위한 모델입니다. ACID는 Atomicity(원자성), Consistency(일관성), Isolation(격리성), Durability(지속성)의 첫 글자를 따온 약어입니다.

  -  Atomicity(원자성)

      원자성은 트랜잭션을 단일 논리적 단위로 간주하여 해당 트랜잭션이 전부 실행되거나 전혀 실행되지 않도록 보장하는 것을 의미합니다. 이러한 원자성은 트랜잭션에 포함된 모든 작업이 성공적으로 완료되지 않으면 이전 상태로 롤백(rollback)됨을 의미합니다.

  - Consistency(일관성)

    일관성은 트랜잭션 전/후 데이터베이스 상태가 일관된 상태를 유지하는 것을 의미합니다. 즉, 트랜잭션이 실행된 후에도 데이터베이스는 일관된 상태를 유지해야 합니다.

  - Isolation(격리성)
    
    격리성은 여러 개의 트랜잭션이 동시에 실행될 때, 각각의 트랜잭션이 서로 영향을 미치지 않도록 보장하는 것을 의미합니다. 즉, 트랜잭션은 다른 트랜잭션에서 변경된 데이터를 참조할 수 없으며, 다른 트랜잭션도 해당 트랜잭션이 변경한 데이터를 참조할 수 없도록 격리되어야 합니다.

  - Durability(지속성)
    지속성은 트랜잭션이 성공적으로 완료되면 해당 결과가 영구적으로 유지되는 것을 보장하는 것을 의미합니다. 즉, 트랜잭션이 완료된 후에도 해당 데이터는 영구적으로 데이터베이스에 유지되어야 합니다.

* SQL에서 RDBMS와 NoSQL의 차이점은 무엇인가요? 어떤 경우에 사용하나요?

    RDBMS는 관계형 데이터베이스 시스템을 말하며, NoSQL은 비관계형 데이터베이스 시스템을 말합니다. 이 둘의 가장 큰 차이점은 데이터 모델링 방식입니다. RDBMS는 정해진 스키마와 테이블을 사용하여 데이터를 구조화하고, 데이터 간의 관계를 테이블 간의 관계로 표현합니다. 반면에 NoSQL은 스키마 없이 데이터를 저장하고, 데이터 간의 관계를 중심으로 저장하지 않습니다.

* SQL에서 인덱스(Index)의 동작 원리는 무엇인가요?

    SQL에서 인덱스는 데이터베이스에서 데이터 검색 속도를 높이기 위해 사용되는 자료구조로, 데이터베이스의 테이블에 대한 검색 속도를 향상시키는 데 사용됩니다. 인덱스는 정렬된 열의 값과 해당 값이 존재하는 데이터 블록의 위치를 포함하는 데이터 구조입니다. 인덱스는 검색 조건에 해당하는 데이터 블록만 스캔하기 때문에 검색 속도가 빨라집니다.
    인덱스는 테이블의 특정 열에 대한 색인으로 생성됩니다. 특정 열이 인덱스로 지정되면 데이터베이스 시스템은 검색 시 해당 열의 값을 기반으로 인덱스를 검색하여 검색 속도를 높입니다. 

* SQL에서 최적화된 쿼리를 작성하기 위한 기본 원칙은 무엇인가요?

    - 적절한 인덱스 사용: 효율적인 쿼리를 작성하려면 적절한 인덱스를 사용해야 합니다. 
    - 적절한 테이블 구조: 테이블 구조를 올바르게 설계하면 쿼리의 성능을 향상시킬 수 있습니다.
    - 적절한 JOIN 사용: JOIN은 데이터베이스에서 가장 느린 작업 중 하나입니다.

* SQL에서 동적 SQL(Dynamic SQL)이란 무엇인가요? 어떤 경우에 사용하나요?

  동적 SQL(Dynamic SQL)은 실행될 때 구문이 생성되는 SQL 쿼리를 의미합니다. 즉, 쿼리가 실행될 때마다 다른 구문이 생성되는 것입니다. 동적 SQL을 사용하는 가장 일반적인 이유 중 하나는 다양한 조건에 따라 동적으로 쿼리를 생성하고 실행해야 할 때입니다.

  동적 SQL을 사용할 때 주의해야 할 점은 SQL 삽입 공격(SQL Injection)을 방지하기 위해 적절한 보안 조치를 취해야 한다는 것입니다.

* 파티셔닝이란 무엇인가요?

   - 파티셔닝(Partitioning)은 대규모 데이터베이스 시스템에서 데이터를 분할하는 기술로, 효율적인 데이터 관리와 검색을 위해 사용됩니다. 파티셔닝은 수직적인(Vertical) 파티셔닝과 수평적인(Horizontal) 파티셔닝으로 나눌 수 있습니다.

    - 수직적인 파티셔닝은 데이터베이스 스키마를 분할하는 것을 의미하며, 테이블의 열(Column)을 분할하는 경우가 있습니다. 수직적인 파티셔닝은 주로 관계형 데이터베이스에서 사용됩니다.

    - 수평적인 파티셔닝은 테이블의 행(Row)을 분할하는 것을 의미합니다. 수평적인 파티셔닝은 대규모 데이터베이스의 성능 향상을 위해 사용되며, NoSQL 데이터베이스에서 많이 사용됩니다.

샤딩은 데이터를 여러 노드에 분산하여 저장하는 기술입니다. 각 노드는 일부 데이터를 담당하며, 데이터의 분산으로 인해 전체 시스템의 성능을 향상시킬 수 있습니다. 샤딩은 수평적인 확장이 가능하며, 데이터의 분산으로 인해 데이터 중복성과 안정성을 보장할 수 있습니다. 하지만, 샤딩을 구현하기 위해서는 데이터베이스의 구조와 코드를 변경해야 하기 때문에 구현이 복잡할 수 있습니다.
